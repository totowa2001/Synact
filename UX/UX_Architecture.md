## 1. 프로젝트 개요

**Synact** — 개인의 고유 행동 데이터로 범용 Action Model을 Fine-Tuning하는 Personal Robotics 플랫폼.

### 핵심 UX 목표

| 우선순위 | 목표 | 설명 |
|---------|------|------|
| ① | **쉬운 파인튜닝** | 비전문가도 자신의 행동에 맞게 로봇을 파인튜닝할 수 있는 직관적 사용성 |
| ② | **하이퍼파라미터 시각화** | 복잡하고 비직관적인 ML 파라미터를 시각적·감성적으로 전달 |

### 기술 백엔드 가정

- **Flow Matching** — 로봇 동작 생성의 핵심 ML 알고리즘
- **모방학습 (IL) + 강화학습 (RL)** 하이브리드 파인튜닝
- **ONNX 기반 모델 추론** (로컬 또는 서버)

```
이름: 김태준, 29세, 현역 펜싱 선수
목표: 세계 최초 '로보틱 펜싱' 종목에서 자신의 펜싱 스타일대로 로봇을 훈련
문제: 로보틱 펜싱은 전례가 없는 동작 — 레퍼런스 데이터가 전무
```

- 반복 훈련 / 점진적 개선 루프

## 3. 새로운 동작을 위한 데이터 수집 방법론

> **핵심 문제:** 로보틱 펜싱처럼 전례 없는 동작은 참조 데이터셋이 없다.
> 단순 촬영 이외의 방법으로 어떻게 "나만의 행동"을 학습 데이터로 만들 것인가?

### 방법 1: 의도 기반 생성 (Intent-Based Prompting)

```
방식: 사용자가 "어깨 높이에서 빠르게 찌르고 즉시 후퇴" 같은
      자연어 또는 키워드로 동작 의도를 입력 →
      시스템이 Flow Matching 기반으로 후보 동작 시퀀스 생성 →
      사용자가 OK/NG 피드백 → 반복 정제

장점: 전례 없는 동작도 언어로 정의 가능
UX 요소: 텍스트 입력창 + 3D 프리뷰 + Approve / Reject 버튼
```

### 방법 2: 비교 선호도 학습 (Comparative Preference, RLHF-style)

```
방식: 시스템이 동일 상황에서 2~3가지 다른 동작 옵션을 생성 →
      사용자가 "이게 더 내 스타일" 선택 →
      선호 패턴 누적으로 개인화 모델 형성

장점: 사용자가 명확히 정의 못해도, 비교를 통해 자신의 스타일 발견
UX 요소: 나란히 재생되는 3D 시뮬레이션 + 선택 버튼 + "왜 이게 더 좋아?" 태그
```

### 방법 3: 키네스테틱 티칭 (Kinesthetic Teaching)

```
방식: 사용자가 로봇의 관절을 손으로 직접 움직이며 동작을 시연 →
      로봇이 관절 센서값 기록 → 이를 학습 데이터로 변환

장점: 언어로 설명하기 어려운 미세한 움직임 캡처 가능
UX 요소: "직접 가르치기 모드" 전환 버튼 + 관절별 기록 타임라인
```

### 방법 4: 레퍼런스 영상 임포트 + 모션 추출

```
방식: 유사한 동작의 영상 (인간 펜싱, 무용, 무술 등) 업로드 →
      자세 추정(Pose Estimation)으로 골격 추출 →
      로봇 관절로 재매핑 후 학습 데이터 생성

장점: 완전히 새로운 동작도 인간 동작에서 참조 가능
UX 요소: 영상 업로드 → 골격 오버레이 프리뷰 → 유사도 슬라이더로 강도 조절
```

### 방법 5: 시뮬레이션 기반 데이터 증강

```
방식: 실제 수집한 소량의 데이터에서 →
      물리 시뮬레이션 환경에서 파라미터 변형 (속도/각도/타이밍) →
      대량의 유사 데이터 자동 생성 (Data Augmentation)

장점: 데이터가 부족한 새 동작의 학습 가능성 확대
파라미터: Time Augmentation (Tj), Trajectory Augmentation (Ti), N값 조절
UX 요소: 증강 양 슬라이더 + 증강 전/후 데이터 분포 시각화
```

### 방법 6: 점진적 목표 정의 (Progressive Goal Shaping)

```
방식: 동작의 "성공 조건"만 정의 (ex. 상대 로봇의 가슴에 닿기) →
      강화학습으로 로봇이 스스로 방법을 탐색 →
      사용자가 결과를 보고 스타일 방향 조율 (너무 공격적/방어적 등)

장점: 완전히 새로운 행동 전략을 로봇 스스로 발견
UX 요소: 성공 조건 정의 카드 + 탐색 진행 실시간 그래프 + 스타일 조절 슬라이더
```

### 방법 7: 단계적 동작 분해 (Skill Decomposition)

```
방식: 복잡한 새 동작을 기존 알려진 하위 동작으로 분해 →
      (ex. 로보틱 펜싱 = 이동 + 균형 유지 + 팔 뻗기) →
      각 하위 동작을 개별 파인튜닝 후 합성

장점: 전례 없는 동작도 기존 역량의 조합으로 근사 가능
UX 요소: 동작 블록 빌더 인터페이스 (Lego처럼 조립)
```

## 4. 정보 아키텍처 (업데이트)

### 기존 IA (원본)

```
로보틱스 연동 → 로보틱스 선택 → 모듈 선택
    → [파인튜닝 → 데이터 선택 → 가중치 조절] → Preview → 모듈 저장
              ↕ 시뮬레이션 (데이터 증강)
```

### 업데이트된 전체 IA

```
[ENTRY]
  0. 스플래시 / 온보딩
      ↓
  1. 홈 대시보드
      ├── 내 로봇 목록
      ├── 진행 중인 파인튜닝
      └── 최근 활동

[SETUP FLOW]
  2. 로보틱스 연동
      ├── 2-1. 로봇 검색 (Bluetooth/WiFi)
      ├── 2-2. 연결 설정 (페어링)
      └── 2-3. 연결 완료 (상태 확인)
          ↓
  3. 로봇 프로필 설정
      ├── 3-1. 기본 정보 (이름, 모델)
      ├── 3-2. 사용 목적 선택 ← [Athlete / Pet / Business]
      └── 3-3. 안전 설정 (동작 범위 제한)
          ↓
  4. 모듈 선택
      ├── 4-1. 모듈 브라우저 (기능 목적별)
      ├── 4-2. 모듈 상세 / 미리보기
      └── 4-3. 기반 모델 선택 (Base Model)

[FINE-TUNING CORE LOOP]
  5. 파인튜닝 허브 (메인)
      ├── 현재 모델 상태
      ├── 학습 진행도
      └── 빠른 실행 버튼
          ↓
  6. 데이터 수집   ←──────────────────────┐
      ├── 6-1. 수집 방법 선택              │
      │     ├── A. 직접 촬영               │
      │     ├── B. 의도 입력 (텍스트)      │
      │     ├── C. 레퍼런스 영상 임포트    │
      │     ├── D. 키네스테틱 티칭         │
      │     └── E. 비교 선호도 선택        │
      ├── 6-2. 수집 실행                   │
      ├── 6-3. 데이터 검토 / 레이블링      │
      └── 6-4. 데이터 라이브러리           │
          ↓                               │
  7. 데이터 선택 (버전)                   │
      ├── 7-1. 데이터셋 목록              │
      └── 7-2. 데이터 비교 뷰             │
          ↓                               │
  8. 하이퍼파라미터 설정 (가중치 조절)    │
      ├── 8-1. 프리셋 (초보자)            │
      └── 8-2. 심화 설정 (전문가)         │
          ↓                               │
  9. 시뮬레이션 (데이터 증강)             │
      ├── 9-1. 시뮬레이션 실행            │
      ├── 9-2. 증강 파라미터 조절         │
      └── 9-3. 결과 확인                  │
          ↓                               │
  10. 프리뷰 (Before / After)            │
      ├── 10-1. 실제 로봇 또는 시뮬       │
      ├── 10-2. 비교 모드                 │
      └── 10-3. 승인 / 재조정 ───────────┘

[SAVE & DEPLOY]
  11. 모듈 저장
      ├── 11-1. 저장 옵션 (로컬/클라우드)
      ├── 11-2. 버전 네이밍
      └── 11-3. 마켓플레이스 공유 (선택)

[LIBRARY & SETTINGS]
  12. 행동 라이브러리
      ├── 저장된 동작 목록
      ├── 동작 편집 / 혼합
      └── 공유 / 다운로드

  13. 설정
      ├── 안전 파라미터
      ├── 계정 / 구독
      └── 연결된 로봇 관리
```

## 5. 사용자 시나리오별 플로우

### 🤺 시나리오 A: 로보틱 펜싱 선수

```
목표: 전례 없는 '로보틱 펜싱' 동작을 자신의 스타일로 파인튜닝

[Day 1 - 초기 설정]
온보딩 → 로봇 연동 → 프로필(목적: 스포츠/Athlete) → 모듈 선택(동작 학습)

[Week 1 - 기초 동작 정의]
데이터 수집(방법: 의도 입력 + 레퍼런스 영상) → 기본 펜싱 자세 시뮬레이션 →
비교 선호도 선택(내 스타일 A vs B) → 하이퍼파라미터(프리셋: 스포츠 정밀) →
시뮬레이션 → 프리뷰 → 저장

[Week 2~4 - 반복 개선]
실제 연습 중 "이 동작이 어색해" 피드백 → 키네스테틱 티칭으로 수정 →
하이퍼파라미터 세밀 조절(Action Chunk Size 증가, Objective Strictness 높임) →
데이터 증강(Time/Trajectory Augmentation) → 재학습 → 프리뷰 → 저장

[경기 전]
행동 라이브러리에서 "공격 스타일" / "수비 스타일" 모듈 전환
```

**핵심 UX 포인트:**
- 수집 방법 선택 화면에서 "새로운 동작 만들기" 탭을 전면에 배치
- 전례 없는 동작임을 인식하고 "탐색 모드"로 자동 전환 안내
- 학습 진행률을 스포츠 훈련 일지 형태로 시각화

---

### 🐴 시나리오 B: 펫 로봇 오너

```
목표: 말 로봇에게 "나만의 성격"과 "나를 알아보는 반응" 부여

[첫 만남]
온보딩 → 로봇 연동 → 프로필(목적: 반려/Pet) → "로봇 이름 짓기"
→ 모듈 선택("반응성 학습" 모듈)

[성격 만들기]
하이퍼파라미터 → 프리셋("친근하게" / "독립적으로" / "장난꾸러기") 선택
※ 기술 용어 없음. 성격 형용사 카드 UI

[행동 가르치기]
데이터 수집(방법: 직접 촬영) → "내가 손 내밀면 다가오기" 패턴 녹화 →
시뮬레이션(즉시 피드백) → 프리뷰 → "맘에 들어요 / 다시 해볼게요"

[공유]
행동 라이브러리 → 마켓플레이스에서 다른 오너의 "귀여운 반응" 템플릿 탐색
```

**핵심 UX 포인트:**
- "파인튜닝"이라는 단어를 사용하지 않고 "길들이기", "가르치기"로 대체
- 하이퍼파라미터는 완전히 숨기고, 미리 정의된 성격 프리셋만 노출
- 모든 피드백은 로봇 캐릭터의 반응으로 의인화

---

### 🏢 시나리오 C: 비즈니스 유저

```
목표: 물류 창고 내 로봇 5대를 동일 작업 특화로 파인튜닝 + 성능 관리

[초기 배포]
온보딩(팀 계정) → 로봇 5대 일괄 연동 → 프로필(목적: Business/물류) →
작업 시나리오 선택(짐 이동 / 경로 최적화)

[데이터 수집]
데이터 수집(방법: 자동화 레퍼런스 + 시뮬레이션 증강) →
5대 동시 데이터 수집 설정 → 데이터 라이브러리(공유 데이터셋)

[파인튜닝 설정]
하이퍼파라미터 → 심화 설정(수치 직접 입력) →
시뮬레이션(5대 동시 시뮬) → 성능 비교 차트

[모니터링 & 개선]
홈 대시보드 → KPI 차트 (작업 효율, 오류율, 학습 진행도) →
성능 저하 로봇 자동 알림 → 재파인튜닝 스케줄

[배포]
저장 → 팀 공유 모듈 → 5대 동시 배포
```

**핵심 UX 포인트:**
- 멀티-로봇 대시보드 뷰 필수
- 성능 지표 (수치 기반) 전면 노출
- 배치 작업 / 스케줄링 UI

### P-00. 스플래시 / 온보딩

**목적:** 처음 사용자에게 Synact가 무엇인지 설명, 페르소나 분기

| 요소 | 내용 |
|------|------|
| **3D 오브젝트** | 말 모양 로봇 3D 모델 — 화면 중앙에 떠 있음, idle 애니메이션 |
| **그래픽** | 배경: 미래적 공간 (파티클 / 그라디언트). 로봇 주변에 행동 궤적 파티클 |
| **텍스트** | "당신의 행동이 로봇을 만든다" / "나만의 로봇, Synact로 시작하세요" |
| **버튼** | [시작하기] — 하단 고정 |
| **플로우** | 스플래시(2초) → 온보딩 슬라이드 3장 → 사용 목적 선택 |

**온보딩 슬라이드 3장:**
1. "로봇은 아직 당신을 모릅니다" — 기본 로봇 행동 시각화
2. "당신의 행동을 보여주세요" — 사람의 동작이 로봇으로 전달되는 애니메이션
3. "이제 당신만의 로봇입니다" — 개인화된 로봇의 반응 시연

### P-01. 사용 목적 선택 (페르소나 분기)

**목적:** 학습 파라미터 설정 — 비전문가는 프리셋, 전문가는 직접 조절

| 요소 | 내용 |

| **효과 미리보기** | 파라미터 변경 시 시뮬레이션 미리보기 자동 업데이트 |
| **리셋 버튼** | [기본값으로 초기화] |

### P-09. 시뮬레이션

**목적:** 실제 로봇 배포 전 시뮬레이션 환경에서 테스트 + 데이터 증강

| 요소 | 내용 |
|------|------|
| **3D 환경** | 시뮬레이션 공간 (배경 선택 가능: 경기장 / 집 / 창고) |
| **로봇 3D** | 시뮬레이션 중 로봇 실시간 움직임 |
| **증강 패널** | 슬라이더 3개: Ti (시간 변형량) / Tj (궤적 변형량) / N (생성 데이터 수) |
| **실시간 차트** | 증강 전/후 데이터 분포 그래프 (실시간 업데이트) |
| **카메라 컨트롤** | 시뮬레이션 뷰어 핀치 줌 / 드래그 회전 |
| **버튼** | [증강 실행] / [다시 실행] / [이 결과로 학습] |

### P-10. 프리뷰 (Before / After)

**목적:** 파인튜닝 전후의 로봇 행동을 비교해 효과 확인

| 요소 | 내용 |
|------|------|
| **레이아웃** | 화면 좌우 또는 위아래 분할 뷰 |
| **Before** | 파인튜닝 이전 기본 모델 동작 |
| **After** | 파인튜닝 후 개인화 모델 동작 |
| **슬라이더** | 중앙 드래그 슬라이더로 Before/After 비율 조절 (와이프 뷰) |
| **차이 강조** | 달라진 관절 움직임을 색상 오버레이로 강조 |
| **평가 UI** | ⭐⭐⭐⭐⭐ 만족도 평가 + "어떤 점이 좋아요?" 태그 |
| **버튼** | [저장하기] / [추가 조절하기 (파라미터로 돌아가기)] / [처음부터 다시] |

---

### P-11. 모듈 저장 및 배포

**목적:** 학습된 모델을 버전으로 저장, 선택적으로 공유

| 요소 | 내용 |
|------|------|
| **버전 카드** | 현재 모델 요약: 데이터 수 / 학습 횟수 / 파라미터 세트 |
| **이름 지정** | "버전 이름" 텍스트 입력 (예: "펜싱 공격 v3") |
| **저장 위치** | 로컬 / 클라우드 선택 |
| **공유 옵션** | [마켓플레이스에 올리기] (선택) → 공개범위 설정 |
| **배포** | Business: [선택한 로봇에 배포] → 대상 로봇 체크리스트 |
| **완료 애니메이션** | 로봇이 새로 배운 동작을 짧게 시연하는 완료 연출 |

---

### P-12. 행동 라이브러리

**목적:** 여러 파인튜닝 모듈을 저장/관리/전환/혼합

| 요소 | 내용 |
|------|------|
| **그리드 뷰** | 저장된 모듈 카드 (이름 / 설명 / 날짜 / 성능 점수) |
| **활성 모듈 표시** | 현재 로봇에 적용 중인 모듈 강조 |
| **혼합 기능** | 두 모듈 선택 → Blending Weight 슬라이더 → 합성 모듈 생성 |
| **즐겨찾기** | 자주 쓰는 모듈 고정 |
| **마켓플레이스** | 다른 사용자의 공개 모듈 탐색 + 다운로드 |
| **롤백** | 이전 버전으로 복구 버튼 (최근 5개 버전 관리) |

---

### P-13. 안전 설정

**목적:** 로봇 동작의 물리적 안전 한계 설정

| 요소 | 내용 |
|------|------|

| **슬라이더** | 최대 속도 / 최대 힘 / 동작 반경 |
| **긴급 정지** | 큰 빨간 버튼 — 항상 최상위 접근 가능 |
| **잠금 설정** | 특정 관절 동작 완전 잠금 (토글) |
| **경고 임계값** | 배터리 / 온도 경고 기준 설정 |

## 8. 하이퍼파라미터 UX 설계

> **설계 원칙:** 모든 파라미터를 "수치" → "경험/비유"로 번역한다.
> 수치는 접어두거나 심화 탭에서만 노출.

---

### HP-1. Learning Rate (학습 민감도)

**비유:** "로봇이 얼마나 빨리 배울까요?"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 거북이 ←────슬라이더────→ 치타 아이콘 |
| **실시간 설명** | 슬라이더 이동 시 하단에 상태 설명 자동 업데이트 |
| **낮음** | "천천히, 꼼꼼하게 배워요. 안정적이지만 시간이 걸려요." |
| **적정** | "균형 잡힌 학습 속도예요." ✓ |
| **높음** | "빠르게 배우지만 혼란스러울 수 있어요. 주의하세요." ⚠️ |
| **수치 노출** | 상세 탭에서만 — 실제 LR 값 표시 |

---

### HP-2. Action Chunk Size (동작 호흡)

**비유:** "한 번에 얼마나 긴 동작을 생각할까요?"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 파동 그래픽 — 짧은 파동 ←→ 긴 파동 슬라이더 |
| **낮음** | "짧게 생각, 빠른 반응. 즉흥적이지만 동작이 끊길 수 있어요." |
| **높음** | "길게 생각, 부드러운 흐름. 느리게 반응하지만 자연스러워요." |
| **시각 비유** | 짧은 박자 vs 긴 호흡의 음악 파형 애니메이션 |

---

### HP-3. Data Augmentation (상상력)

**비유:** "부족한 데이터를 얼마나 상상으로 채울까요?"

**3가지 하위 파라미터의 통합 UI:**

```
┌─────────────────────────────────────────┐
│  상상력 설정                             │
│                                         │
│  시간 변형 (Ti)    ○━━━━━━━━━━━━━━━○     │
│  "비슷한 타이밍의 동작을 만들어요"        │
│                                         │
│  경로 변형 (Tj)    ○━━━━━━━━━━━━━━━○     │
│  "비슷한 궤적의 동작을 만들어요"         │
│                                         │
│  생성 양 (N)       ○━━━━━━━━━━━━━━━○     │
│  "원본 1개로 몇 개를 만들까요?" 0x ~ 20x │
│                                         │
│  예상 생성 데이터: 원본 47개 → 940개    │
└─────────────────────────────────────────┘
```

| 상태 | 설명 |
|------|------|
| 낮음 | "원본에 충실해요. 특수한 동작에 적합." |
| 적정 | "부족한 데이터를 잘 보완해요." ✓ |
| 높음 | "많이 만들지만 원본과 달라질 수 있어요." ⚠️ |

---

### HP-4. Context Dependence (상황 감수성)

**비유:** "로봇이 상황 변화에 얼마나 민감하게 반응할까요?"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 레이더 차트 — 원이 클수록 다양한 상황 대응 |
| **낮음** | "좁은 상황에서만 특기를 발휘해요. 특수 목적에 적합." |
| **높음** | "다양한 상황에 유연하게 반응해요. 하지만 특수성이 줄어들어요." |

---

### HP-5. Loss Weight Balancing (따라 하기 vs 스스로 배우기)

**비유:** "선생님 흉내 내기 vs 스스로 터득하기"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 양쪽에 추가 달린 저울 그래픽 |
| **왼쪽 (IL 높음)** | "선생님(내 동작)을 정확히 따라해요. 예측 가능하지만 경직될 수 있어요." |
| **오른쪽 (RL 높음)** | "스스로 방법을 찾아요. 창의적이지만 예상 밖 행동이 나올 수 있어요." |
| **Pet 언어** | "엄마 말 잘 듣기 ←→ 자기 방식대로" |

---

### HP-6. Blending Weights (행동 혼합 비율)

**비유:** "여러 동작을 어떻게 섞을까요?"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 파이 차트 / 컬러 혼합 슬라이더 (행동 A의 색 + 행동 B의 색) |
| **예시** | "공격 스타일 70% + 수비 스타일 30%" |
| **적용** | 슬라이더 조절 → 3D 프리뷰에서 즉시 혼합 결과 확인 |

---

### HP-7. Objective Strictness (목표 지향 vs 방식 복제)

**비유:** "결과가 같으면 되나요, 방법까지 똑같아야 하나요?"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 과녁 아이콘 ← 슬라이더 → 발자국 아이콘 |
| **낮음 (방식 복제)** | "내가 한 것처럼 똑같이 움직여요. 행동이 일관적이에요." |
| **높음 (목표 지향)** | "'목적'을 향해가는 방법을 스스로 찾아요. 유연하지만 예측이 어려워요." |

---

### HP-8. Que Threshold (자율 판단력)

**비유:** "로봇이 얼마나 자신 있을 때 행동할까요?"

| 시각화 | 내용 |
|--------|------|
| **위젯** | 뇌/신뢰도 게이지 (Confidence Meter) |
| **낮음** | "잘 모르는 상황에서도 배운 대로 행동해요. 적극적이지만 오작동 가능." |
| **높음** | "아주 확실한 상황에서만 행동해요. 안전하지만 소극적." |
| **Pet 언어** | "용감한 편 ←→ 신중한 편" |

---

### 프리셋 정의

| 프리셋 이름 | 대상 | 파라미터 세트 설명 |
|------------|------|-----------------|
| 🏃 빠른 학습 | 데이터 충분, 빠른 결과 원할 때 | LR 높음, Chunk 중간, Aug 낮음 |
| 🎯 정밀 학습 | 스포츠/정밀 작업 | LR 낮음, Objective Strictness 높음 |
| 🐣 안전 학습 | 첫 파인튜닝, 비전문가 | 모든 파라미터 보수적, Que Threshold 높음 |
| 🎨 창의 학습 | 새로운 동작, Pet 개성 | RL 비율 높음, Context Dependence 높음 |
| 💼 대량 학습 | Business, 데이터 부족 | Data Augmentation 최대, LR 중간 |

## 부록: 핵심 UX 원칙 요약

1. **기술 → 경험 번역** — 모든 ML 용어는 사용자 경험 언어로 번역
2. **페르소나별 적응** — 동일 기능을 3가지 언어와 복잡도로 제공
3. **즉각적 시각 피드백** — 파라미터 변경 → 즉시 3D 프리뷰 반영
4. **점진적 노출** — 기본 → 프리셋 → 심화 설정 순서로 복잡도 전환
5. **되돌리기 보장** — 모든 단계에서 롤백 가능, 실험적 탐색 장려
6. **전례 없는 동작 지원** — "새 동작 만들기" 플로우를 1등 시민으로 취급
7. **로봇을 캐릭터로** — 로봇을 UI의 중심 3D 오브젝트로 배치, 상태 의인화
